# AGENTS.md

## Scope

These instructions apply to all files under `docs/`.

## Documentation Rules

- Use English file names.
- Use kebab-case for Markdown files, for example `technical-design-and-test-cases.md`.
- Keep document titles descriptive and human-readable.
- Prefer Chinese prose for product and engineering documents unless the user asks otherwise.
- Use stable headings so sections can be linked and reviewed easily.
- Keep product scope and engineering design in separate documents.
- Include test cases in engineering documents when a design affects implementation behavior.
